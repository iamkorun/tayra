tayra burrows into your git history, reads conventional commits, and sniffs out the right version bump in milliseconds.

📖

Reads Conventional Commits

Parses feat fix chore docs and more. Detects breaking changes via ! or BREAKING CHANGE:.

🤖

Built for CI/CD

--ci outputs just the version string. Drop it into GitHub Actions, GitLab CI, or any shell script.

🏷️

Auto-detects tag style

Using v1.2.3 or bare 1.2.3? tayra detects your prefix and preserves it.

⚡

Zero config, instant

No YAML, no .tayrarc. Run tayra in any git repo and get an answer in milliseconds.

Dig into CI mode

One flag, machine-readable output. Pipe it straight into your release script.

CI pipeline
$ tayra --ci
v1.5.0
$ git tag $(tayra --ci) && git push --tags
# Tag created and pushed in one line

How bumps are decided

tayra picks the highest bump level across all commits since the last tag.

Commit type Bump
fix: chore: docs: refactor: ... patch
feat: minor
feat!: fix!: BREAKING CHANGE: major
Highest bump level wins. One feat! = major, regardless of other commits.
